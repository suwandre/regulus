# regulus M1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the runnable regulus core — set CPU+GPU power limits with a software combined budget, show live telemetry, switch persisted profiles, reapply on boot/wake — on the reference machine (Windows · AMD 9955HX3D · RTX 5090 Laptop), behind a HAL that makes other backends additive.

**Architecture:** Single elevated process. A pure-logic core (`regulus-hal`, `regulus-engine`) defines HAL traits, the capability model, budget math, and profile/persistence logic — fully unit-tested with mock backends. A Windows backend crate (`regulus-backend-windows`) implements the traits against real hardware (NVML for GPU, libryzenadj FFI for CPU power, ring0 MSR reads for CPU telemetry). A Slint app crate (`regulus-app`) renders a capability-driven UI (tray quick-panel + window) bound to engine state.

**Tech Stack:** Rust (workspace), Slint (UI, software renderer first), `nvml-wrapper` (GPU), `libloading` (libryzenadj DLL FFI), ring0 driver for MSRs (spike), `serde`/`toml` (config), `tray-icon` (tray), `windows` crate (power events, Task Scheduler, elevation).

## Task tags

Every task is tagged `[AUTO]` (testable without hardware → runs in the Ralph build/review/repair loop) or `[HW]` (requires the reference machine → human-in-the-loop), plus `model·effort`.

**Methodology note — why HW tasks differ in shape:** `[AUTO]` tasks use strict TDD (failing test → code → green → commit) with complete code. `[HW]` tasks that probe unknown hardware behavior (e.g. whether an MSR responds) are **spikes**: a goal, an exact validation procedure, and a decision gate — not a pre-written test, because the open question *is* whether the hardware cooperates. This is intentional, not a placeholder. HW tasks still end with committed code and a recorded result.

//! Core domain types shared across the HAL.

use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Serialize, Deserialize)]
pub struct Watts(u32);

impl Watts {
    pub const fn new(w: u32) -> Self { Watts(w) }
    pub const fn get(self) -> u32 { self.0 }
    pub const fn as_milliwatts(self) -> u32 { self.0 * 1000 }
    pub fn clamp(self, lo: Watts, hi: Watts) -> Watts {
        Watts(self.0.clamp(lo.0, hi.0))
    }
}

/// CPU power limits in the AMD SMU model (sustained / slow PPT / fast PPT).
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub struct CpuPowerLimits {
    pub stapm: Watts,
    pub slow: Watts,
    pub fast: Watts,
}

/// Discrete GPU power-limit target (TGP).

/// One telemetry sample. Fields are Option because not every backend can read every metric

    pub gpu_temp_c: Option<f32>,

#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum Capability {

    GpuPower,
    Telemetry,
    Cooler,
    Display,
    Overclock,

    pub fn insert(&mut self, c: Capability) { self.0.insert(c); }

//! Software combined-budget policy: split a total CPU+GPU budget into per-domain
//! caps. These are caps, not guaranteed draw. Minimums are hard floors (a domain
//! is never starved below its minimum even if that exceeds the total target).

use regulus_hal::types::Watts;

#[derive(Debug, Clone, Copy)]
pub struct DomainLimits {
    pub cpu_min: Watts,
    pub cpu_max: Watts,
    pub gpu_min: Watts,
    pub gpu_max: Watts,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub struct Split {
    pub cpu: Watts,
    pub gpu: Watts,
}

/// `bias`: 0.0 = all CPU, 1.0 = all GPU. Clamped to [0,1].
pub fn split_budget(total: Watts, bias: f32, limits: &DomainLimits) -> Split {
    let total = total.get() as i64;
    let bias = bias.clamp(0.0, 1.0) as f64;

    let (cpu_min, cpu_max) = (limits.cpu_min.get() as i64, limits.cpu_max.get() as i64);
    let (gpu_min, gpu_max) = (limits.gpu_min.get() as i64, limits.gpu_max.get() as i64);

    // Raw desired split.
    let mut gpu = (total as f64 * bias).round() as i64;
    let mut cpu = total - gpu;

    // Clamp each to its range.
    gpu = gpu.clamp(gpu_min, gpu_max);
    cpu = cpu.clamp(cpu_min, cpu_max);

    // If we're under target (a domain clamped low), hand surplus to the other domain
    // up to its max. Bias direction decides who gets first refusal.
    let mut leftover = total - (cpu + gpu);
    if leftover > 0 {
        if bias >= 0.5 {
            let give = leftover.min(cpu_max - cpu);
            cpu += give; leftover -= give;
            let give = leftover.min(gpu_max - gpu);
            gpu += give;
        } else {
            let give = leftover.min(gpu_max - gpu);
            gpu += give; leftover -= give;
            let give = leftover.min(cpu_max - cpu);
            cpu += give;
        }
    }

    Split {
        cpu: Watts::new(cpu.max(0) as u32),
        gpu: Watts::new(gpu.max(0) as u32),
    }
}
```

//! Persistence of profiles + the active selection. Default config dir is
//! %APPDATA%\regulus\config.toml; load_from/save_to take an explicit path for tests.
use crate::profile::Profile;
use serde::{Deserialize, Serialize};
use std::path::{Path, PathBuf};

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct StoredState {
    pub active: String,
    pub profiles: Vec<Profile>,
}

impl Default for StoredState {
    fn default() -> Self {
        StoredState { active: "Balanced".into(), profiles: Profile::presets() }
    }
}

pub fn config_path() -> PathBuf {
    let base = std::env::var("APPDATA").unwrap_or_else(|_| ".".into());
    PathBuf::from(base).join("regulus").join("config.toml")
}

pub fn load_from(path: &Path) -> Result<StoredState, String> {
    match std::fs::read_to_string(path) {
        Ok(s) => toml::from_str(&s).map_err(|e| e.to_string()),
        Err(_) => Ok(StoredState::default()),
    }
}

pub fn save_to(path: &Path, state: &StoredState) -> Result<(), String> {
    if let Some(parent) = path.parent() {
        std::fs::create_dir_all(parent).map_err(|e| e.to_string())?;
    }
    let s = toml::to_string_pretty(state).map_err(|e| e.to_string())?;
    std::fs::write(path, s).map_err(|e| e.to_string())
}
```

use regulus_hal::traits::{HalResult, PowerControl};
use regulus_hal::types::{CpuPowerLimits, GpuPowerLimit, Watts};

/// Map a CPU sustained budget into STAPM/slow/fast. M1 policy: slow = +15%, fast = +30%,
/// each clamped to cpu_max by the split's domain limits upstream.
fn cpu_limits_from_sustained(stapm: Watts, cpu_max: Watts) -> CpuPowerLimits {
    let s = stapm.get();
    CpuPowerLimits {
        stapm,
        slow: Watts::new(((s as f32 * 1.15) as u32).min(cpu_max.get())),
        fast: Watts::new(((s as f32 * 1.30) as u32).min(cpu_max.get())),
    }
}

pub fn apply_profile(
    power: &mut dyn PowerControl,
    profile: &Profile,
    limits: &DomainLimits,
) -> HalResult<()> {
    let split = split_budget(Watts::new(profile.budget_watts), profile.bias, limits);
    power.set_cpu_limits(cpu_limits_from_sustained(split.cpu, limits.cpu_max))?;
    power.set_gpu_limit(GpuPowerLimit(split.gpu))?;
    Ok(())
}
```

//! Telemetry poller. The engine calls `tick` on a 1–2 Hz timer (real timer lives in
//! the app; tests drive tick directly). UI reads `latest` — never the hardware.
use regulus_hal::traits::Telemetry;
use regulus_hal::types::TelemetrySample;

#[derive(Default)]
pub struct Poller {
    latest: TelemetrySample,
}

impl Poller {
    pub fn new() -> Self { Self::default() }
    pub fn latest(&self) -> TelemetrySample { self.latest }
    pub fn tick(&mut self, source: &mut dyn Telemetry) {

    }

//! Reapply controller: limits reset on reboot/wake, so re-push the active profile
//! when the OS signals boot or resume. The app feeds OS power events in here.
use crate::budget::DomainLimits;
use crate::engine::apply_profile;
use crate::profile::Profile;
use regulus_hal::traits::PowerControl;

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PowerEvent { Boot, Resume, Suspend }

pub struct ReapplyController {
    active: Profile,
    limits: DomainLimits,
    reapply_count: u64,
}

impl ReapplyController {
    pub fn new(active: Profile, limits: DomainLimits) -> Self {
        Self { active, limits, reapply_count: 0 }
    }
    pub fn set_active(&mut self, p: Profile) { self.active = p; }
    pub fn reapply_count(&self) -> u64 { self.reapply_count }

    pub fn on_event(&mut self, power: &mut dyn PowerControl, ev: PowerEvent) {
        if matches!(ev, PowerEvent::Boot | PowerEvent::Resume) {
            if apply_profile(power, &self.active, &self.limits).is_ok() {
                self.reapply_count += 1;
            }
        }
    }
}

//! GPU backend via NVML (nvml-wrapper). Power limit is set in milliwatts.
use nvml_wrapper::Nvml;
use regulus_hal::traits::{HalError, HalResult};
use regulus_hal::types::{GpuPowerLimit, Watts};

pub struct GpuNvml { nvml: Nvml, index: u32 }

impl GpuNvml {
    pub fn new() -> HalResult<Self> {
        let nvml = Nvml::init().map_err(|e| HalError::Backend(e.to_string()))?;
        Ok(Self { nvml, index: 0 })
    }
    pub fn set_limit(&self, limit: GpuPowerLimit) -> HalResult<()> {
        let dev = self.nvml.device_by_index(self.index).map_err(|e| HalError::Backend(e.to_string()))?;
        dev.set_power_management_limit(limit.0.as_milliwatts())
            .map_err(|e| HalError::Backend(e.to_string()))
    }
    pub fn range(&self) -> HalResult<(Watts, Watts)> {
        let dev = self.nvml.device_by_index(self.index).map_err(|e| HalError::Backend(e.to_string()))?;
        let c = dev.power_management_limit_constraints().map_err(|e| HalError::Backend(e.to_string()))?;
        Ok((Watts::new(c.min_limit / 1000), Watts::new(c.max_limit / 1000)))
    }
    pub fn read(&self) -> HalResult<(Watts, f32, u32)> {
        let dev = self.nvml.device_by_index(self.index).map_err(|e| HalError::Backend(e.to_string()))?;
        let power = dev.power_usage().map_err(|e| HalError::Backend(e.to_string()))? / 1000;
        let temp = dev.temperature(nvml_wrapper::enum_wrappers::device::TemperatureSensor::Gpu)
            .map_err(|e| HalError::Backend(e.to_string()))? as f32;
        let clock = dev.clock_info(nvml_wrapper::enum_wrappers::device::Clock::Graphics)
            .map_err(|e| HalError::Backend(e.to_string()))?;
        Ok((Watts::new(power), temp, clock))
    }
}
```

```rust
//! CPU power backend via libryzenadj.dll (FFI). We deliberately avoid the power-metric
//! table init that crashes on Fire Range; only the SMU setters are called.
use libloading::{Library, Symbol};
use regulus_hal::traits::{HalError, HalResult};
use regulus_hal::types::CpuPowerLimits;

type RyzenAccess = *mut std::ffi::c_void;

pub struct CpuRyzenAdj { lib: Library, ry: RyzenAccess }

impl CpuRyzenAdj {
    pub fn new(dll_path: &str) -> HalResult<Self> {
        unsafe {
            let lib = Library::new(dll_path).map_err(|e| HalError::Backend(e.to_string()))?;
            let init: Symbol<unsafe extern "C" fn() -> RyzenAccess> =
                lib.get(b"init_ryzenadj").map_err(|e| HalError::Backend(e.to_string()))?;
            let ry = init();
            if ry.is_null() { return Err(HalError::Backend("init_ryzenadj returned null".into())); }
            Ok(Self { lib, ry })
        }
    }

    fn set_one(&self, sym: &[u8], value_mw: u32) -> HalResult<()> {
        unsafe {
            let f: Symbol<unsafe extern "C" fn(RyzenAccess, u32) -> i32> =
                self.lib.get(sym).map_err(|e| HalError::Backend(e.to_string()))?;
            let rc = f(self.ry, value_mw);
            if rc != 0 { return Err(HalError::Backend(format!("{} rc={}", String::from_utf8_lossy(sym), rc))); }
            Ok(())
        }
    }

    pub fn set_limits(&self, l: CpuPowerLimits) -> HalResult<()> {
        self.set_one(b"set_stapm_limit", l.stapm.as_milliwatts())?;
        self.set_one(b"set_slow_limit",  l.slow.as_milliwatts())?;
        self.set_one(b"set_fast_limit",  l.fast.as_milliwatts())?;
        Ok(())
    }
}

impl Drop for CpuRyzenAdj {
    fn drop(&mut self) {
        unsafe {
            if let Ok(cleanup) = self.lib.get::<unsafe extern "C" fn(RyzenAccess)>(b"cleanup_ryzenadj") {
                cleanup(self.ry);
            }
        }
    }
}
```
